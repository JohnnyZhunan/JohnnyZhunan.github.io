Unity总结（三）

  • 2018年4月
  • 2018-04-29

C# Serializable(序列化）

如果只从Unity的使用以及直观的从Unity界面的角度讲，建议看看这个链接：
Unity中的Serializable

但是从C#语言角度讲，建议大家看看这个链接，对于小白来讲这个解释比较专业化，建议大家慢慢理解：
C# Serializable对象序列化的作用

SerializeField与Serializable之间的关系

建议参考这个链接，我认为比较直观且好理解：
SerializeField和Serializable

按键事件

在C#当中，若是想得到按键触发的效果，可以使用GetKey……系列方法，具体方法参照这个链接：
Unity键盘、鼠标，检测按键输入事件

但是这个当中，涉及一个一个问题：键盘码（KeyCode），网上有不少链接，都把编号什么的给出来了，但是，在具体操作中，什么按键对应什么样的KeyCode，建议大家看这个链接：
unity3d KeyCode各键值说明
而对于KeyCode对照表，建议大家看这个链接：
KeyCode对照表

Input输入事件

参考这个链接：
关于Unity中的Input输入事件
不过上述链接只对键盘事件做出了解释，那么如何对相关组件（不仅仅是Input事件）进行实时监听（类似于前端当中的OnChange），参考这个链接：
NGUI OnChange Event

C#当中的var类型

直观从用法上，不知道能否理解成语C++中的auto类型类似。用法以及相关的注意事项参考这个链接：
C#中的隐式类型var——详细示例解析

C#泛型类（Dictionary）

简而言之，是一种映射关系，具有一一对应的关系，具体通过一种什么方式对应呢？答案是哈希表,下面给出两个链接，一个介绍了Dictionary，另一个介绍了哈希表：
Dictionary 泛型类.aspx)

哈希表（散列表）原理详解

Unity当中的Camera

要看最详解，还是要看官网，官网上的描述非常详尽，只是官网的速度比较慢：
Camera

本人在这周的工作当中遇到的一个比较烦的问题是，根据窗口界面动态调整Camera的Viewer Port Rect，它到底是什么东西，参考官方文档：
Rect
只是官网貌似说的有问题，Rect的第一个和第二个参数，貌似应该是左下角，而不是官网所给的左上角。

Layers

原理的话，还是推荐官方网站：
Layers
在使用Layer的时候，也应注意起与LayerMask的区别，关于这个，参考这个链接：
浅谈Unity3D中Layers和LayerMask
